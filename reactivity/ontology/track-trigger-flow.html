Vue3 Track/Trigger 机制流程图

Track（依赖收集）流程

flowchart LR A[读取响应式属性] --> B[触发 getter] B --> C{判断是否应该收集} C -->|是| D[track target, key] C -->|否| E[直接返回] D --> F[查找/创建 Dep] F --> G[trackEffects dep] G --> H[dep.add activeEffect] H --> I[effect.deps.push dep] I --> J[返回属性值] E --> J style A fill:#42b883,color:#fff style D fill:#d69e2e,color:#fff style F fill:#3182ce,color:#fff style H fill:#805ad5,color:#fff

Trigger（触发更新）流程

flowchart LR A[属性赋值] --> B[触发 setter] B --> C[trigger target, key, newVal, oldVal] C --> D[查找 Dep] D --> E{Dep 存在?} E -->|否| F[直接返回] E -->|是| G[合并相关 Dep] G --> H[ITERATE_KEY 数组length] H --> I[triggerEffects 合并后的Dep] I --> J{区分 effect 类型} J -->|computed| K[先执行 computed] J -->|普通 effect| L[执行普通 effect] K --> M[标记 dirty 为 false] L --> N[执行 effect 函数] M --> O[完成] N --> O style A fill:#e53e3e,color:#fff style C fill:#d69e2e,color:#fff style I fill:#805ad5,color:#fff style K fill:#38a169,color:#fff style L fill:#3182ce,color:#fff
